## 🔄 工作流

这批书的 PDF 都是**文字层 + 扫描版混合**的，`pdf2m` 完美处理：

```bash
# 1. 假设 OCR 工具（MinerU）把结果放在 _mineru_batch_v2/ 下
# 2. 一行命令批量转换
pdf2m ./_pdfs \
  --ocr-dir ./_mineru_batch_v2 \
  -o ./_md/v2 \
  -w 4
```

实际输出（截取 `01_庄子集解内篇补正_王先谦.md` 前 30 行）：

```markdown
# 庄子集解内篇补正

## 第 1 页

鹏之背，不知其几千里也。
```

## 🎯 关键效果

- **章节识别**：自动识别 `内篇·逍遥游第一` 等章名 → H3
- **书名识别**：自动识别 `庄子集解内篇补正` 为书名 → H1
- **页标记**：每页以 `## 第 N 页` 分隔
- **段落合并**：古文段落正确识别断句，不被切碎
- **页码过滤**：自动剔除 `庄子集解 ........ 12` 之类的页眉

## 💡 为什么这个工具能搞定古籍？

古籍 PDF 通常是**双层混排**（影印底图 + 不可复制文字），
而 `pdf2m` 的设计核心是：

```
文字层 PDF  →  PyMuPDF 直接抽（最准）
扫描版      →  OCR 工具预处理，再让 pdf2m 合并
任意来源    →  启发式合并，结构化输出
```

无需为古籍单独写规则，通用启发式就能很好工作。
